背包问题主要有 0-1背包（每种物品只有一个），完全背包（每种物品可以无限拿），多重背包（同种物品有有限多个）三种。

完全背包

个人感觉完全背包最简单，上代码

看到这个问题，就觉得它与切钢条问题很像：对于长为l的钢条的切割最优，其子结构k与子结构l-k也必然最优
不过完全背包在取得某个最优时，背包可能有剩余空间。
则再用一个表rest_vol[k]记录容量k取得最优时的剩余空间，
然后对容量i进行二分时，把剩余容量也考虑在内best_value[k]+best_value[i-k+rest_vol[k]]
自底向上刷 best_val 和 rest_vol，再补上一点细节，感觉挺妥的。

是不是与硬币找零问题也有点像？硬币找零问题简直就是完全背包的简化情况

matplotlib

Posted on 2018-12-21 |

matplotlib 很综合，所以它的api也有些复杂。以 matplotlib.pyplot 为例，它主要有两个level的api：

pyplot 的两个api level

这两个 level api 功能大致一样，可以互相转换。对于一般的需求，官方document 的意思是推荐 object level

pyplot level

At this level, simple functions are used to add plot elements (lines, images, text, etc.) to the current axes in the current figure

object level

At this level, the user uses pyplot to create figures, and through those figures, one or more axes objects can be created. These axes objects are then used for most plotting actions.

Again, for these simple examples this style seems like overkill, however once the graphs get slightly more complex it pays off.
